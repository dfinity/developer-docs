You have deployed your first canister. Here is where to go based on what you want to build.

## Choose your path

**I want to understand what makes ICP different.**
Explore the [Concepts](/concepts/chain-key-cryptography/) section to learn about chain-key cryptography, [HTTPS outcalls](/concepts/https-outcalls/), [VetKeys](/concepts/vetkeys/), and other features that set the Internet Computer apart.

**I want to build a full-stack dapp.**
The [Building Apps](/guides/canisters/lifecycle/) section walks you through canister development, frontend integration, authentication, storage, and deployment patterns.

**I want to connect to Bitcoin or Ethereum.**
[Chain Fusion](/concepts/chain-fusion/) covers cross-chain integrations: ckBTC, ckETH, EVM RPC calls, and threshold signatures for signing transactions on other chains.

**I want to build DeFi.**
The [DeFi](/reference/token-standards/) section covers token standards, ledger canisters, chain-key tokens, and Rosetta API integration.

**I want to launch a token.**
Start with [Token Standards](/reference/token-standards/) to understand ICRC-1 and ICRC-2, then learn how to deploy and manage ledger canisters.

**I want to decentralize governance of my project.**
[Governance](/concepts/governance/) explains the SNS (Service Nervous System) framework for launching a DAO that controls your dapp.

**I want to learn Motoko or Rust for ICP.**
The [Languages](/languages/motoko/) section has language-specific guides, patterns, and best practices for canister development.

## Essential tools

- [icp-cli documentation](https://dfinity.github.io/icp-cli/) -- guides, references, and CLI commands
- [icskills](https://skills.internetcomputer.org) -- AI agent skills for ICP development
- [IC Dashboard](https://dashboard.internetcomputer.org/) -- monitor canisters, transactions, and network health
- [Mops](https://mops.one/) -- Motoko package manager
- [Crates.io](https://crates.io/search?q=ic-cdk) -- Rust crates for ICP (`ic-cdk`, `candid`, etc.)
