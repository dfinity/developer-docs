Internet Identity (II) is the ICP-native authentication system. It uses WebAuthn/FIDO2 (passkeys, hardware keys) to provide secure, privacy-preserving authentication without passwords. Each user gets a unique principal per application, preventing cross-app tracking.

## Authentication flow

1. **User visits your app** and clicks "Sign in with Internet Identity"
2. **Your app** creates an `AuthClient` and opens the II frontend (`https://identity.ic0.app`)
3. **II frontend** authenticates the user via WebAuthn (passkey, fingerprint, hardware key)
4. **II canister** creates a delegation chain: a short-lived session key signed by the user's identity
5. **Your app** receives the delegation and uses it to sign canister calls

```
User → Your App → II Frontend → II Canister (rdmx6-jaaaa-aaaaa-aaadq-cai)
                                      │
                                      ▼
                              Delegation Chain
                              (session key + expiry)
                                      │
                                      ▼
                              Your App uses delegation
                              to make authenticated calls
```

## Principal derivation

Internet Identity derives a unique principal for each (user, application) pair:

```
Principal = DER(SHA-224(user_number || frontend_origin || ii_canister_id))
```

This means:
- The same user gets a **different principal** for each app they use
- Apps cannot correlate users across origins
- The user's anchor number is never revealed to the application

## Delegation chains

A delegation is a signed statement: "This session key can act on behalf of this principal until expiry."

| Field | Description |
|-------|-------------|
| `pubkey` | The session public key (generated by your app) |
| `expiration` | Nanosecond timestamp when the delegation expires |
| `targets` | Optional: restrict delegation to specific canister IDs |

Delegations can be chained. The II canister signs the first delegation, and your app can create sub-delegations for specific canisters.

## Anchor management

Each user has an **anchor** (numeric identity) that groups their authentication devices:

- Passkeys (platform authenticators — fingerprint, Face ID)
- Hardware security keys (YubiKey, etc.)
- Recovery phrases (BIP-39 seed phrases)

Users can add or remove devices from their anchor at any time through the II frontend.

## Integration

For a practical guide to integrating Internet Identity in your app, see [Internet Identity](/guides/authentication/internet-identity/).

The II canister is deployed at `rdmx6-jaaaa-aaaaa-aaadq-cai`. For local development, deploy a local II instance:

```bash
icp deploy internet-identity --environment local
```
